# 10 Tailwind CSS Tips for Better Productivity

Tailwind CSS has revolutionized how we write CSS. Here are 10 tips to level up your Tailwind game.

## 1. Use @apply for Repeated Patterns

Instead of repeating utility classes, create custom components:

```css
@layer components {
  .btn-primary {
    @apply px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600;
  }
}
```

## 2. Customize Your Config

Extend Tailwind's default theme:

```javascript
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: '#your-color'
      }
    }
  }
}
```

## 3. Use Arbitrary Values

For one-off values, use square brackets:

```html
<div class="top-[117px]">
```

## 4. Group Hover States

Apply hover effects to children:

```html
<div class="group">
  <p class="group-hover:text-blue-500">Hover me</p>
</div>
```

## 5. Responsive Design First

Mobile-first approach:

```html
<div class="w-full md:w-1/2 lg:w-1/3">
```

## 6. Dark Mode Support

Enable dark mode in your config:

```javascript
module.exports = {
  darkMode: 'class'
}
```

## 7. Use Plugins

Extend functionality with official plugins:

- @tailwindcss/typography
- @tailwindcss/forms
- @tailwindcss/aspect-ratio

## 8. Optimize for Production

Tailwind automatically purges unused CSS in production builds.

## 9. Use JIT Mode

Just-In-Time mode generates styles on-demand for faster builds.

## 10. Learn the Class Order

Follow a consistent order:
1. Layout (flex, grid)
2. Spacing (margin, padding)
3. Sizing (width, height)
4. Typography
5. Visual (background, border)
6. Misc (cursor, pointer-events)

## Conclusion

These tips will help you write cleaner, more maintainable Tailwind CSS code. Happy styling!
